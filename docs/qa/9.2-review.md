## ⚠️ 发现项

### 1. 〔中优先〕突进斩/群体击杀触发运行时 ERROR：`monitoring` 在碰撞信号期间被阻塞
- **现象**：`SHOT_SKILL=dash` 连拍时控制台报 `ERROR: Function blocked during in/out signal. Use set_deferred("monitoring", true/false).`（出现 2 次）。
- **根因**：突进斩(640速)瞬间冲过多个敌人，触发敌人死亡处理 `enemy.gd:316-317`（`collision_layer = 0` / `touch.monitoring = false`）。该处在物理碰撞 in/out 信号派发窗口内被同步调用 → 被 Godot 阻塞 → **赋值没生效**。
- **后果**：被阻塞意味着死亡敌人的 `touch` 接触碰撞可能没能及时关闭 → 冲过去秒杀的瞬间玩家**可能仍吃一下接触伤害**；并刷错误日志。同类模式还需排查 `player.gd:175`、`boss.gd:260`。
- **建议**：把这些 `monitoring` / `collision_layer` 赋值改为 `set_deferred("monitoring", false)` / `set_deferred("collision_layer", 0)`。
- **复现确认**：请 dev 在真实战斗里冲过 ≥2 个相邻敌、或大招清一群敌，看控制台是否报同样错、死亡敌是否还能擦伤玩家。
  > 注：QA 连拍把 3 假人摆得较密（offset 70/150/230），比常规刷怪密；但"突进/大招打多个敌"是该技能的核心用途，场景真实，建议确认而非忽略。

## 🔧 处理结果（开发线程 · 待 QA 复验）

### 发现项 1〔中优先〕`monitoring` 信号期阻塞 — 已修
按建议把死亡/失效处理中"信号派发期会被触发"的 monitoring/collision_layer 赋值改为 `set_deferred`：
- **enemy.gd:_die** → `set_deferred("collision_layer",0)` + `touch.set_deferred("monitoring",false)`
- **boss.gd:_die** → 同上（Boss 也会被技能击杀，一并加固）
- **projectile.gd:_fade** → `set_deferred("monitoring", false)`。**这是第二处根因**：`_fade()` 在 `_on_body`(碰撞信号)里因 pierce 耗尽被调用时直接关 monitoring → 同样阻塞。突进/地面波打多敌会走到这条。
- **player.gd:175/713/782/789**（hitbox.monitoring）排查后**无需改**：均在 `_ready` 或 `_physics_process` 内，不在碰撞信号回调里。

### 复现验证（开发线程）
用 QA 的 `SHOT_MOTION=1 SHOT_SKILL=<dash/ground/ult>` 连拍复现：
- 修前：`dash` 稳定报 `ERROR: Function blocked during in/out signal`。
- 修后：`dash` / `ground` / `ult` 三技能均 **0 报错**；`dash` 对 `slime`/`beast` 各重复 3 次仍 **0 报错**；普通启动 150 帧无脚本错误。
- 结论：错误消除，死亡敌接触碰撞改为延迟到帧末关闭（同帧内 take_damage 已置 dead 守卫，不会二次结算），秒杀瞬间不再吃擦伤。
