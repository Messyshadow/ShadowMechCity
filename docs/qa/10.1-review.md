## 🔧 本阶段顺带修复的潜伏 bug（高价值）
- **门换房在碰撞信号期重建几何 → `Can't change this state while flushing queries`**：`_on_door`(body_entered 信号) 原**直接**调 `_enter_room`→`_make_oneway` 等在物理 flush 期改碰撞 → 报错。**这是影响游戏内每次过门的潜伏 bug**，此前 grep 太窄漏掉；本次 SHOT 默认落点正好在下行门上把它放大成 100+ 才暴露。
- **修法**：`_on_door` 改 `_enter_room.call_deferred(...)`（并先置 door_cd 防重入）。修后逐房 0 报错。
- 新增 `hazard.gd` 用 `_physics_process` 轮询扣血(不改物理状态)，无 flush 风险。

## 诚实边界（待人工试玩）
- **可达性**：以实拍帧 + 几何核算确认（平台间距在双跳/冲刺范围内，上行门有攀爬栈）；但**未真人走通全程**——位移手感/某些跳跃是否偏难需人工试玩。
- **难度/数值平衡**（敌人 HP/伤害、陷阱伤害、精英强度）godot-capture 测不出 → 待人工试玩调。
- 陷阱"蒸汽阀"目前是常驻判定(带冷却)，非"周期喷发"节奏；如需更像机关可后续做开合周期。

## 🎨 设计质量迭代（回应 `docs/qa/10.1-design-feedback.md`）

按"设计质量红线"P1~P5 重做四房布局 + 三个新机制：

### 新机制（新增脚本）
- **`mover.gd`** 移动平台/链条升降（AnimatableBody2D + sync_to_physics，载玩家）；`movers` 字段 `[cx,cy,w,axis,dist,period,phase]`。
- **`belt.gd`** 传送带（Area2D 轮询把站立玩家横向位移）；`belts` 字段 `[cx,cy,w,h,push]`。
- **`updraft.gd`** 热气流（Area2D 轮询托举玩家上行）；`updrafts` 字段。
- **`pits`** 字段：地板开坑 + 坑底"接住实体 + 熔铁危害"——掉坑=危害但可跳出，**跨坑走平台/链条=必经动线**（P1）。
- **蒸汽阀改周期喷发 + 预警**（`hazard.gd`：预警闪烁→喷高柱有伤→收，气柱判定在喷口上方）（P4）。

### 四房改动（兑现招牌=P5 / 连动线=P2）
- **熔炉大厅**：中央**巨型高炉**实体塔（翻越）+ **热气流**柱托举上行 + 左侧**熔铁池**开坑；平台连成"跨池→上行→翻炉→右门"动线，高处放宝箱逼上行。
- **装配车间**：**传送带**流水线（左推/右推）+ 底部双坑 + 齿轮移到坑上窄道（看时机过）；精英重装蒸汽兵。
- **熔铁回廊**：贯穿中部的**熔铁河**（宽坑伤3）+ **断桥**平台跨越 + 河上齿轮；左侧攀爬栈到上行门；暗墙后生命碎片。
- **传送链区**：**链条升降机**（垂直 mover 上行门）+ **链条平台**（水平 mover 跨双坑）+ 冲刺门到 Boss。

### 自测（playbook 全错误扫描 + 实拍帧）
- 全 ERROR 扫描：导入/启动150帧/**四房逐一加载 = 0**。
- 实拍宽景（Read PNG `_r_factory_*`）：确认 熔铁河/熔铁池开坑、热气流柱、传送带箭头、链条平台轨道、新铸造厂背景 均渲染；地板被坑/高炉切断，平台成必经。

### 诚实边界（待人工试玩）
- **移动平台载人/链条升降的乘坐手感**、热气流托举力度、传送带速度、坑宽与断桥间距 → 静态帧测不出，**需人工试玩调**。
- **防软锁已保底**：所有坑都有"坑底接住实体"（掉坑=熔铁伤害但能跳出远侧），即使乘坐机制手感不佳也不会卡死；且 Boss 另有 熔炉大厅→熔铁回廊→传送链区 的通路，不单依赖热气流翻高炉。
