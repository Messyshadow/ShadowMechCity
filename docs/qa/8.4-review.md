1. **不重生逻辑正确**：`main.gd:_enter_room` 仅当 `not Game.is_collected(sc[3])` 才刷出 secret → 已收集不再出现。`collect_secret` 对已收集 id 直接 return，不重复计数。
2. **永久 +1 最大生命**：`player.max_hp()` 加上 `Game.heart_pieces`；拾取顺序正确——先 `collect_secret`(heart_pieces++) 再 `heal(99)`，补满到**新的**更高上限。
3. **存档完整**：`collected` + `heart_pieces` 写入 save / 读档 / reset 三处都覆盖。
4. **3 碎片唯一 id + 门控正确**（截图确认刷出与摆放）：
   - `heart_cavern_climb` `(110,120)`：cavern 左上攀墙高台（原宝箱已替换为碎片）。见 `screenshots/qa3_cavern_shard.png`

   - `heart_temple` `(90,500)`：temple 炸弹暗墙 `[200,-40,40,600]` 后，需炸弹。见 `screenshots/qa3_temple_shard.png`
   - 三 id 唯一，`HEART_TOTAL=3` 对应。
5. **地图收集度**（代码确认）：`map_panel` 绘制"已探索区域 X/N" + "生命碎片 X/3"，满收集变金色。

## ⚠️ 发现项

### 1. 生命碎片颜色非"金色"、且与装备(gear)同贴图同形状（低优先 · 辨识度）
- 现象：三个房间截图里碎片均渲染为**红橙色菱形**，并非代码注释期望的金色 `(1.0,0.85,0.35)`。原因：`modulate` 与基础贴图 `shard.png`（本身偏红）相乘，乘不出金色。非光照问题（冷蓝的 hub 里同样红橙）。
- 隐患：**生命碎片(heart)与装备掉落(gear)都用同一张 `shard.png`**，形状相同、颜色又都可能偏暖/偏红（高稀有度 gear 也偏金红）。玩家可能把"重要的永久升级碎片"误认成普通装备，或反之。
- 建议（择一，非阻塞）：① 给生命碎片用**独立贴图**（心形/独特轮廓）最稳；② 或换成基础为白/灰的贴图再 modulate，确保金色真的金，并与所有 gear 稀有度色明显区别。功能不受影响，可留待阶段 13（美术统一）一起做。

### 2. 探索度分母待自查（极低优先）
`map_panel` 用 `Rooms.ROOMS.size()` 作"已探索 X/N"的分母。建议开发线程确认 `ROOMS` 里没有非真实房间的条目混入分母（当前看是 hub/temple/mine/depths/tunnel/cavern/mine_boss 共 7 间，正常）。
