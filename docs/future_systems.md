# 创意系统设计（未来阶段开发用）

> 用户构思的 4 个核心特色系统，互相关联，构成"暗影机械城"的机械朋克特色玩法。
> 记录于设计阶段，待后续阶段实现。与技能树（skill_tree.md）深度结合：每个系统对应新的技能分支/加点。

---

## 系统一：机器人召唤系统（Robot Summon）
- 可召唤**无人机（空中机器人）**和**地面机器人**进行战斗与辅助。
- 成长：初始**只能召唤 1 个**机器人作战；随该能力不断升级强化，最多可**同时召唤 3 个不同机器人**协同作战。
- 机器人类型设想：空中无人机（远程/侦查）、地面机器人（近战/坦克/辅助）等，各有定位。
- 与「虚拟量子空间」联动：召唤的机器人来自量子空间中存放/培养的机器人。

## 系统二：装备升级系统（Equipment Upgrade）
- 装备可像技能/角色一样**升级改造**。
- 世界观设定：不同装备（武器/防具等）出自**不同公司**，或**同公司不同产品线**；
  通过**卫星储能 + 量子技术**精准投射到用户硬件上完成升级——
  把现实中"软件升级补丁 + 个性化主题/订阅"的模式，套用到**硬件装备**上。
- 消耗：每次升级/改造**消耗金币**，花费根据**装备品质等级**与**升级能力**不同而不同（品质越高/能力越强越贵）。
- 与「装备/背包系统」（inventory_system.md）结合：升级即提升装备词条/属性 + 外观个性化。

## 系统三：黑客入侵系统（Hacking）
- 后期解锁的黑客技能，针对敌方机器人：
  - **破坏核心**：让机器人持续掉血。
  - **临时编程为友军**：一定时间内变成友军助战（**有时间限制，且不可跟随**）。
  - **制造混乱**：让机器人互相攻击 / 乱射。
- 难度机制：**敌人等级越高，网络渗透越难**入侵；需不断**强化黑客技能等级**。
- 风险：入侵**比自己更高级别**的敌人，**可能反受伤害**（反噬）。

## 系统四：虚拟量子空间系统（Virtual Quantum Space）
> 类比《宝可梦》精灵中心的存放空间，但内部是 **2D 横版平面**，机器人可在其中自由活动。

- **两类空间**（多层结构，类似宝可梦存放中心分层）：
  - **活动空间**：非战斗状态开启，角色会进入空间内活动；可像宝可梦一样
    给机器人**投放道具升级 / 学习技能**进行培养。**战斗状态下不可开启**。
  - **战斗空间**：可在战斗中开启使用，但**有冷却时间**。
    - 可设置**先进环境** / **降低敌人属性与机动性**的环境，由**玩家加点**形成。
    - 可在战斗中将**敌方机器人转化收入**到活动空间，供日后召唤（联动系统一）。
- 收服机制（参考宝可梦）：
  - 敌人**实力/等级越强越难收**；**血量越高越难收，血量越低越好收**。
  - 敌人**等级越强**，在战斗空间中**控制他的时间越短**；空间时间结束后只能等冷却再开。

## 系统关联图
```
[虚拟量子空间] --存放/培养--> [机器人] --召唤--> [战斗]
      ^                                            |
      |--战斗空间"收服"敌方机器人------------------|
[黑客入侵] --临时策反/破坏--> 敌方机器人
[装备升级] --强化--> 角色/武器/防具(金币消耗)
```

## 实现优先级建议（后续阶段插入点）
- 这些属于**特色玩法层**，建议在"游戏外壳/存档"(阶段4)、"Boss"(阶段5)、"装备系统"(阶段6) 之后，
  作为阶段7+ 的**特色系统**逐个落地：
  - 7a 机器人召唤（基础：召唤1个→3个）
  - 7b 虚拟量子空间（活动空间培养 + 战斗空间收服）
  - 7c 黑客入侵
  - 7d 装备升级（卫星量子投射改造）
- 每个系统都新增对应**技能树分支**（召唤/黑客/空间/改造）。
