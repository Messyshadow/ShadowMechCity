# 区域设计模板（阶段 10 各区域通用标准）

> 用途：阶段 10 做每个区域（10.1~10.6）都照此建，确保"大、迷宫、多层、银河城感"，不再是小走廊。
> 配套：`docs/design/地图设计指引.md`（策划草图原则）、`docs/DEV_PLAN.md`（各区域具体敌人/Boss）。
> 拍板（2026-06-27 头脑风暴）：**区域结构 = 几个大迷宫房间连成群**。

## 一、区域结构（已拍板）

**一个区域 = 4~6 个"大迷宫房间"用门互联**（沿用现有多房间 + 门系统，不是单一超大房）。
- 不要 1 个区域只做 1~2 个小房间。
- 房间之间多向互联（左右上下门 + 区域内传送门），形成环路而非单线。

## 二、每个"大迷宫房间"的标准（核心，别再做空走廊）

照「地图设计指引」第 2、3 点：
1. **更大**：bounds 比现状明显加大（现状多为 1300~1700 宽；新房建议 1800~2600 宽，并加高做多层）。
2. **内部迷宫**：房间内部用隔墙/平台段构成**绕行/分层路径**，而非一条平地。用 `platforms`/`oneways` + 内部实体墙（`_make_solid` 竖墙段）堆出迂回。
3. **多层立体**：上中下多层，高低错落，需跳跃/冲刺/二段跳/攀墙/滑翔穿行（正好复用 8.x 能力）。
4. **多出入口**：一个大房可有多个门/传送门（草图：三角落各一）。
5. **死路与隐藏龛**：内部留小龛、隐藏房入口、宝箱/生命碎片位（接 8.4 收集系统 `secrets`）。
6. **细节物件**：管道/齿轮/灯光/背景层次，去"空旷感"。

## 三、银河城门控与动线

- 每个区域至少 1 个**能力门**结构化动线（DEV_PLAN 指定，如 10.4 矿区=冲刺门）。临界路径 vs 回溯支线分明。
- **能力门拾取物本身不能要求该能力**才能到（防"鸡生蛋"软锁——见 8.3 验收教训）。
- 区域内放 1 个**生命碎片/收集隐藏点**（`secrets`，需某能力或炸暗墙），鼓励回溯。
- Boss 房用**封闭竞技场**（已有挡门机制），击败给奖励 + 解锁后续。

## 四、敌人与难度

- 用该区域**专属敌种**（DEV_PLAN 每区域已列）+ 至少 1 个**精英**变体（更高 HP/新行为）。
- 难度沿动线递增；新手区（中央车站）不刷敌。
- 可加**陷阱/环境危害**（熔岩、毒液、齿轮、强风——各区域主题相关）。

## 五、地图与完成度

- 房间填 `map: Vector2i(x,y)` 在世界地图正确分块；多层用上中下分层（map_system.md）。
- 隐藏房/收集计入完成度统计（接 8.4 地图收集度）。

## 六、技术纪律（别再踩物理坑）

- 任何在碰撞/信号回调或物理 flush 期改物理状态（`monitoring`/`collision_layer`/`disabled`/`add_child` 带碰撞体节点）一律 `set_deferred`/`call_deferred`（见 9.2/9.3 验收）。
- 做完按 `docs/qa/QA_PLAYBOOK.md` 自测：godot-capture 取景拍各房间布局/能力门可达性 + **全 ERROR 扫描** + 写 `docs/qa/10.x-review.md`。

## 七、每区域交付清单（逐项打勾）

- [ ] 4~6 个大迷宫房间，互联成环（非单线）
- [ ] 每房：更大 + 内部迷宫 + 多层 + 多出入口 + 隐藏龛 + 细节物件
- [ ] ≥1 能力门结构化动线（拾取物不软锁）
- [ ] 1 个生命碎片/收集隐藏点
- [ ] 专属敌种 + ≥1 精英 + 主题陷阱
- [ ] Boss 封闭竞技场 + 奖励 + 解锁
- [ ] 世界地图分块/分层正确，隐藏房计入完成度
- [ ] godot-capture 验布局/可达 + 全 ERROR 扫描 0 + 写验收报告
